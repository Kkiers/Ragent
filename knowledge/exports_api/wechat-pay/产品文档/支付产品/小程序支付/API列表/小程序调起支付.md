# 小程序调起支付

路径：产品文档/支付产品/小程序支付/API列表/小程序调起支付

通过[JSAPI/小程序下单](https://pay.weixin.qq.com/doc/v3/merchant/4012791897)接口获取到发起支付的必要参数prepay_id，然后使用微信支付提供的[小程序方法](https://developers.weixin.qq.com/miniprogram/dev/api/payment/wx.requestPayment.html)调起微信支付。

## 接口说明

支持商户：【普通商户】

## 字段说明

### 调用wx.requestPayment(OBJECT)发起微信支付

接口名称： wx.requestPayment，详见[小程序API文档](https://developers.weixin.qq.com/miniprogram/dev/api/payment/wx.requestPayment.html)

`timeStamp` 必填 string(32)

【时间戳】标准北京时间，时区为东八区，自1970年1月1日 0点0分0秒以来的秒数。
注意：部分系统取到的值为毫秒级，商户需要转换成秒(10位数字)。

`nonceStr` 必填 string(32)

【随机字符串】不长于32位。该值建议使用随机数算法生成。

`package` 必填 string(128)

【预支付交易会话标识】[JSAPI/小程序下单](https://pay.weixin.qq.com/doc/v3/merchant/4012791897)接口返回的`prepay_id`参数值，提交格式如：`prepay_id=***`

`signType` 必填 string(32)

【签名类型】默认为RSA，仅支持RSA。

`paySign` 必填 string(512)

【签名值】使用字段appId、timeStamp、nonceStr、package计算得出的签名值，详细参考：[小程序调起支付签名](https://pay.weixin.qq.com/doc/v3/merchant/4012365341)。

注意：此处签名需使用实际调起支付小程序`appid`，且为[JSAPI/小程序下单](https://pay.weixin.qq.com/doc/v3/merchant/4012791897)时传入的`appid`，微信支付会校验下单与调起支付所使用的`appid`的一致性。

### 回调结果

用户从微信收银台返回商户小程序时会触发商户调用requestPayment拉起支付时传入的function回调函数，商户可通过回调errMsg参数展示相应支付结果。

注：前端回调并不保证它绝对可靠，不可只依赖前端回调判断订单支付状态，订单状态需以后端[查询订单](https://pay.weixin.qq.com/doc/v3/merchant/4012791900)和[支付成功回调通知](https://pay.weixin.qq.com/doc/v3/merchant/4012791902)为准。

| 回调类型 | errMsg | 说明 |
| --- | --- | --- |
| success | requestPayment:ok | 调用支付成功 |
| fail | requestPayment:fail cancel | 用户取消支付 |
| fail | requestPayment:fail (detail message) | 调用支付失败，其中 `detail message` 为后台返回的详细失败原因，如果错误原因为"errno": 102, "errMsg": "requestPayment:fail jsapi has no permission，说明该小程序使用支付功能的权限已被公众平台限制，请前往[公众平台](https://mp.weixin.qq.com)登录后查看相关通知 |

# 公益补助

路径：产品文档/运营工具/商家转账/附录/转账场景报备信息字段传参说明/公益补助

## 1. 概述

当商户发起转账场景为公益补助时，需通过发起转账接口中传入以下参数。其中，transfer_scene_id参数需传入对应场景ID；user_recv_perception参数可选择传入对应的用户收款感知；此外，还需通过transfer_scene_report_infos参数传入转账场景报备信息，用于报备该笔转账资金的转账背景。

transfer_scene_report_infos为数组类型参数，在公益补助的转账场景下需固定传两条明细，每条“转账场景报备信息明细”包含info_type、info_content两个参数。

## 2.转账场景ID传值要求

需在发起转账接口通过transfer_scene_id【转账场景ID】参数传入对应的场景ID，此参数值为固定值，请传入：1013

## 3.用户收款感知传值要求

需在发起转账接口通过user_recv_perception【用户收款感知】参数传入对应的收款感知，当前仅支持传入【公益补助金】；若未传入该参数，默认展示为【公益补助金】。

## 4.转账场景报备信息参数传值要求

需在发起转账接口通过transfer_scene_report_infos【转账场景报备信息】参数传入对应的报备信息，以报备该笔转账资金的转账背景。

transfer_scene_report_infos参数传值要求：

| 字段简称 | 字段名 | 类型[长度限制] | 是否必填 | 示例值 | 描述 | 传值示例 |
| --- | --- | --- | --- | --- | --- | --- |
| 信息类型 | info_type | string [1, 15] | 必填 | 公益活动名称 | 信息类型，两条明细中必须分别填写以下两个取值：公益活动名称公益活动备案编号注：信息类型不可重复填写，请严格按照示例传入 | "transfer_scene_report_infos" :  [{    "info_type" :   "公益活动名称",    "info_content" : "在民政部的备案名称"},{    "info_type" :   "公益活动备案编号",    "info_content" : "在民政部的备案编号"}] |
| 信息内容 | info_content | string [1, 32] | 必填 | 在民政部的备案名称 | 信息内容，请填写在民政部的备案名称 |  |

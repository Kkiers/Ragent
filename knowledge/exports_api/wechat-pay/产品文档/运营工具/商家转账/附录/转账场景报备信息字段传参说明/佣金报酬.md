# 佣金报酬

路径：产品文档/运营工具/商家转账/附录/转账场景报备信息字段传参说明/佣金报酬

## 1. 概述

当商户发起转账场景为佣金报酬时，需通过发起转账接口中传入以下参数。其中，transfer_scene_id参数需传入对应场景ID；user_recv_perception参数可选择传入对应的用户收款感知；此外，还需通过transfer_scene_report_infos参数传入转账场景报备信息，用于报备该笔转账资金的转账背景。

transfer_scene_report_infos为数组类型参数”，在佣金报酬的转账场景下需固定传两条明细，每条“转账场景报备信息明细”包含info_type、info_content两个参数。

## 2.转账场景ID传值要求

需在发起转账接口通过transfer_scene_id【转账场景ID】参数传入对应的场景ID，此参数值为固定值，请传入：1005

## 3.用户收款感知传值要求

需在发起转账接口通过user_recv_perception【用户收款感知】参数传入对应的收款感知，当前仅支持传入【劳务报酬】、【报销款】、【企业补贴】、【开工利是】，不传则默认展示【劳务报酬】。

## 4.转账场景报备信息参数传值要求

需在发起转账接口通过transfer_scene_report_infos【转账场景报备信息】参数传入对应的报备信息，以报备该笔转账资金的转账背景。

transfer_scene_report_infos参数传值要求：

| 字段简称 | 字段名 | 类型[长度限制] | 是否必填 | 示例值 | 描述 | 传值示例 |
| --- | --- | --- | --- | --- | --- | --- |
| 信息类型 | info_type | string [1, 15] | 必填 | 岗位类型 | 信息类型，两条明细中必须分别填写以下两个取值：岗位类型报酬说明注：信息类型不可重复填写，请严格按照示例传入 | "transfer_scene_report_infos" :  [{    "info_type" :   "岗位类型",    "info_content" : "外卖员"},{    "info_type" : "报酬说明",    "info_content" : "7月份配送费"      }] |
| 信息内容 | info_content | string [1, 32] | 必填 | 外卖员 | 信息内容，商户自定义内容，如外卖员 |  |
